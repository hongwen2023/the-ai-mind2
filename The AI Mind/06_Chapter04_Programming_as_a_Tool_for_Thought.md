# 第 4 章 · 编程作为思维工具

在第 3 章的末尾，我们把计算的铁律看清楚了：有些问题不可判定，有些问题的代价随规模指数爆炸，而多项式与指数之间横着一道质的鸿沟。那一章是"知"——知道计算能做什么、要付出什么代价。这一章是"行"——学会**亲手驾驭**它。

但我要在第一段就把一件事讲死，免得你带着错误的期待读下去：**这不是一章 Python 语法教程。** 你不会在这里学到 `for` 的语法或者列表怎么切片——那些东西任何一本入门书都讲得比我细。我们要建立的是一种更根本的东西：一种**思维方式**。做 AI 的人和一般写程序的人，在"如何用代码思考"这件事上，分岔得很早、很深。这一章就是要把那个分岔点讲清楚，并让你站到正确的那一侧。它是全书工程线的第一块肌肉。

## 一、概念主线：代码是可以被执行的思想

先纠正一个几乎人人都有的默认假设。大多数人心里，编程的位置是这样的：**先在脑子里想清楚，然后把想清楚的东西"翻译"成代码。** 想是思考，写代码是苦工。代码是思想的下游产物，是把已经成型的念头落实成机器能跑的形式。

对做 AI 的人来说，这个假设是错的，而且是有害的错。

真实的情形是：**代码是思考本身的媒介。** 你脑子里那个念头——"把这些数据标准化一下""这两组样本的距离应该这样算"——在你把它写成能跑的代码之前，它是模糊的。它模糊到你自己都没意识到有多模糊。你以为你想清楚了"标准化"，直到你不得不回答机器逼你回答的问题：是按行还是按列求均值？标准差用 $n$ 还是 $n-1$ 做分母？某一列恰好是常数、标准差为零时除法怎么办？这些问题不是实现细节，它们**就是**"标准化"这个概念的真实内容。你在脑子里可以对它们视而不见；机器不行。请记住这几个具体的问题——本章的代码会一个一个地把它们逼到你面前，让你亲眼看看"被逼精确"是什么样子。

所以真正的工作循环是这样一个环，而不是一条直线：

> 把一个念头写成能跑的代码 → 跑它 → 看结果 → 修正念头 → 再写。

注意这个环的每一圈都在改变你脑子里的东西，不只是改变屏幕上的字符。你写出来、跑一遍、发现输出的形状是 $(6, 3)$ 而你以为该是 $(3, 6)$，于是你对"我到底在算什么"的理解就前进了一格。**代码逼你把模糊的直觉精确到机器能执行的程度**，而这个"被逼精确"的过程，本身就是最好的思考训练——它不允许你在任何一处含糊过关。

这正是 Prelude 里"术语的幻觉"的工程版本。当时我们说：你能说出"梯度下降"四个字，不等于你懂梯度下降；真懂的标志之一，是你能让它在机器上跑通、跑对。**能跑通才算真懂**——不是因为跑通是懂的证明，而是因为逼自己跑通的那个过程，会把"术语的幻觉"里所有你以为已经填上、其实还是空白的地方，一个一个照出来。代码是一台"幻觉检测仪"。

带着这个视角，本章要完成一次具体的心智转变，它是 AI 编程区别于一般编程最锋利的一刀：

> **从"一次处理一个元素"转向"一次操作整块数据"。**

这就是**向量化 (vectorization)** 的思维。一般程序员想到"给一千个数每个都加一"，脑子里浮现的是一个循环：走到第一个，加一；走到第二个，加一……做 AI 的人脑子里浮现的是**一个动作**：给这一整块数据加一。前者把数据看成一列需要逐个搬运的砖头，后者把数据看成一个可以被整体操纵的对象。这两种"看"，会长成两种完全不同的头脑。本章剩下的部分，就是把这一刀磨利。

## 二、数学线（The Why）：张量是装数据的容器

要一次操作整块数据，你首先得有"整块数据"这个东西的名字和形状。在 AI 里，它叫**张量 (tensor)**，而它其实就是"多维数组"的另一个说法。让我们从最小的单位往上搭：

- **标量 (scalar)**：一个数，$x \in \mathbb{R}$。形状是"空"，$0$ 维。比如一个损失值。
- **向量 (vector)**：一列数，$\mathbf{x} \in \mathbb{R}^{d}$。$1$ 维，形状 $(d,)$。比如一个样本的 $d$ 个特征。
- **矩阵 (matrix)**：一张表，$X \in \mathbb{R}^{n \times d}$。$2$ 维，形状 $(n, d)$。比如 $n$ 个样本、每个 $d$ 个特征——**这正是第 1 章里那个 $X$**。
- **高维张量**：$3$ 维、$4$ 维……比如一批 RGB 图片是 $(N, C, H, W)$：$N$ 张图、$C$ 个颜色通道、高 $H$、宽 $W$。

请把这条链子和第 1 章接起来。第 1 章我们把"学习"形式化时，数据用 $X$、$Y$ 指代，那时它们是数学符号。现在我们要问一个工程问题：**怎么把 $X$ 装进机器？** 答案就是张量。张量是把"数据"这个抽象概念装进内存的**容器**。这一节你只需要记住这件事，因为第 5 章我们会专门问"该往这个容器里装什么、世界怎么变成这些数字"——那时容器已经就位。

### 向量化的数学实质

现在讲一句本章数学线的核心判断：**那些你想用逐元素循环写出来的东西，绝大多数其实是在表达线性代数运算。** 循环只是你用低级语言把线性代数一个字母一个字母拼出来而已。

看几个例子。两个向量的**点积 (dot product)**：

$$\mathbf{a} \cdot \mathbf{b} = \sum_{i=1}^{d} a_i b_i.$$

你可以写成 `s = 0; for i: s += a[i]*b[i]`。但那个求和号 $\sum$ 已经在告诉你：这是一个整体操作，`a @ b` 就是它。**矩阵-向量乘** $\mathbf{y} = A\mathbf{x}$，$y_i = \sum_j A_{ij} x_j$，看似两层循环，其实一个 `A @ x`。**逐元素运算 (element-wise)**，比如两个同形张量相加 $C_{ij} = A_{ij} + B_{ij}$，`A + B` 一步到位。

真正需要多讲两句直觉的是**广播 (broadcasting)**。它回答的问题是：形状不一样的两个张量，怎么做逐元素运算？比如一个 $(n, d)$ 的矩阵，要给它**每一行**都减去同一个 $(d,)$ 的均值向量。逐元素运算本来要求形状严格相同，但广播允许机器"假装"把那个 $(d,)$ 的向量沿着行的方向复制 $n$ 份，凑成 $(n, d)$，再逐元素相减——请注意"假装"这两个字：被拉伸的那个操作数并不真的在内存里复制 $n$ 份，机器只是在计算时按需读取它的同一份数据。（这一点稍后会有一个重要的但书，别急。）

广播的规则可以精确到一句话：**把两个形状右对齐，逐维比较；每一维要么相等，要么其中一个是 $1$（就沿那一维拉伸到另一个的长度），否则报错。** 举例：

$$(6, 3) \ \text{与}\ (3,) \ \xrightarrow{\text{右对齐}}\ (6,3)\ \text{与}\ (1,3)\ \Rightarrow\ (6,3).$$

$(3,)$ 在左边补一个 $1$ 维变成 $(1,3)$，然后 $1 \to 6$ 拉伸，结果 $(6,3)$。这正是"每行减均值"。再看一个更漂亮的：$(3,1)$ 和 $(1,4)$，两边各自拉伸，得到 $(3,4)$——这就是外积式扩张的形状。稍后代码里我们会亲眼验证这两个形状。

### 复杂度视角：诚实地说清向量化赢在哪

这里必须泼一盆冷水，因为很多材料在这里含糊其辞，甚至骗人。**向量化通常不改变大 O 复杂度。** 给一个长度 $n$ 的向量每个元素加一，循环版是 $O(n)$，向量化版还是 $O(n)$——机器底层照样要碰 $n$ 个数。标准化 $n \times d$ 的矩阵，两版都是 $O(nd)$。它们**同阶**。

那向量化赢在哪？赢在**常数因子**，以及**表达的清晰度**。常数因子的来源是实打实的工程事实：向量化调用底下是编译好的 C / Fortran（BLAS 库）在跑，一条指令能用 SIMD 同时处理多个数（single instruction, multiple data），数据在内存里连续存放、缓存命中率高（内存局部性）。相比之下，Python 解释器每绕一圈循环都要做一堆装箱拆箱、类型检查的杂事。同样是 $O(nd)$，前面那个 $O$ 里藏着的常数可以差出一两个数量级。

请把这句话和第 3 章的铁律接起来，因为这是本章最需要智识诚实的地方：**向量化是常数因子上的胜利，不是复杂度级别的胜利。它翻不过第 3 章那堵指数墙。** 一个 $O(2^n)$ 的算法，向量化之后还是 $O(2^n)$，只是让你在墙根下多走几步、慢一点撞上去。指数爆炸该炸还是炸。向量化是把该做的活儿做得又快又干净的手艺，不是绕过计算铁律的魔法。把这两件事分清楚，你才不会在该换算法的时候去优化常数，也不会在该向量化的时候误以为问题出在复杂度。

### 顺带埋一颗种子：数值不是数学

最后点一句、不展开：机器里的浮点数 (floating point) 不是数学里的实数。它们有有限精度，$0.1 + 0.2$ 不精确等于 $0.3$。所以稍后我们比较两版结果时，用的不是 `==` 而是 `np.allclose`——"足够接近"而非"完全相等"。还有一件事：许多计算依赖随机数，而要让结果可复现，就得**固定随机种子 (random seed)**。这两件事——浮点的不精确、随机的可复现——是数值思维的种子，第 9 章讲数值稳定性时会长成大树。这里只需要你记住它们存在。

## 三、工程线（The How）：把同一件事写两遍

现在到本章的主线。我们要做一件看起来多此一举、实则是全章灵魂的事：**把同一个计算写两遍**，一遍用纯 Python 循环，一遍用向量化的 NumPy，然后**断言两者结果一致**。

为什么值得写两遍？因为这正是第一节那句话的现场：**同一个想法，两种表达。** 循环版是你"一次处理一个元素"的头脑的样子，向量化版是你"一次操作整块数据"的头脑的样子。把它们并排放，你就能亲眼看见那次心智转变。而 `np.allclose` 的断言，顺手就是一个**回归测试**——将来你重构向量化版，循环版就是它诚实的裁判。

我们选的计算是**数据标准化 (standardization)**：把每一列减去该列均值、再除以该列标准差，让每列变成均值 $0$、标准差 $1$。这是几乎每个 AI 流程开头都要做的事，而且它天然是一个广播的例子（每行减去均值向量）。

在动手前，先兑现第一节抛出的那个问题——**标准差的分母用 $n$ 还是 $n-1$？** 我们两版都选 $n$（总体标准差，NumPy 里叫 `ddof=0`）。这不是随手一选：一来它让两版的分母严格对齐，等价断言才有意义；二来这与生产环境里 `sklearn` 的 `StandardScaler` 一致。选 $n-1$（无偏样本标准差）会得到略微不同的数，那个取舍留到第 9 章。**注意这就是"被逼精确"的现场**：脑子里的"标准化"三个字对分母是沉默的，代码逼你把它说死。

下面是从上到下可运行、只依赖 `numpy`、固定了随机种子的主脚本：

```python
import numpy as np

rng = np.random.default_rng(42)                 # 固定种子：抽样序列完全确定
X = rng.normal(loc=5.0, scale=2.0, size=(6, 3)) # 6 个样本, 3 个特征
n, d = X.shape

# ---------- 版本 A：纯 Python 循环（一次一个元素）----------
means = [0.0] * d
for j in range(d):                        # 逐列求均值
    s = 0.0
    for i in range(n):
        s += X[i, j]
    means[j] = s / n
stds = [0.0] * d
for j in range(d):                        # 逐列求标准差（分母 n，即 ddof=0）
    s = 0.0
    for i in range(n):
        s += (X[i, j] - means[j]) ** 2
    stds[j] = (s / n) ** 0.5

Z_loop = [[0.0] * d for _ in range(n)]
n_scalar_loop = 0
for i in range(n):
    for j in range(d):                    # 逐元素标准化
        Z_loop[i][j] = (X[i, j] - means[j]) / stds[j]
        n_scalar_loop += 1                # 只数这一步的标量赋值
Z_loop = np.array(Z_loop)

# ---------- 版本 B：向量化（一次一整块）----------
mu = X.mean(axis=0)        # (3,)  逐列均值；一行顶掉上面第一个双层 for
sigma = X.std(axis=0)      # (3,)  逐列标准差；默认 ddof=0（分母 n），与循环版对齐
Z_vec = (X - mu) / sigma   # 广播: (6,3)-(3,) -> (6,3)
n_array_ops_vec = 4        # 整个标准化用了 4 个数组级操作：mean、std、减、除

# ---------- 断言：两个想法的两种表达，结果必须一致 ----------
assert np.allclose(Z_loop, Z_vec)
print("allclose:", np.allclose(Z_loop, Z_vec))
print("循环版 Python 层标量赋值 (仅标准化那步):", n_scalar_loop)
print("向量化版 Python 层数组级操作 (mean/std/减/除):", n_array_ops_vec)
print("标准化后列均值≈0:", np.allclose(Z_vec.mean(axis=0), 0.0))
print("标准化后列标准差:", np.round(Z_vec.std(axis=0), 6))
```

运行这段脚本，你应当看到：

```
allclose: True
循环版 Python 层标量赋值 (仅标准化那步): 18
向量化版 Python 层数组级操作 (mean/std/减/除): 4
标准化后列均值≈0: True
标准化后列标准差: [1. 1. 1.]
```

逐行把代码连回概念。`X.mean(axis=0)` 里的 `axis=0` 说的是"沿着第 0 个轴（行的方向）塌缩"，塌缩掉 $6$ 个样本，剩下 $(3,)$ 的每列均值——它一行就干完了循环版里那整个双层 `for`。`(X - mu) / sigma` 这一行是全脚本的心脏：`X` 是 $(6,3)$，`mu` 是 $(3,)$，广播规则把 `mu` 右对齐、沿行拉伸成 $(6,3)$，于是**每一行都减去了同一个均值向量**——这就是第二节那条广播规则的现场。

现在看证据。**我们不拿计时当主要证据**，因为计时会随机器、负载起伏，不确定、不可断言。我们拿**操作次数**当证据，它确定、可复现、可断言——但要诚实地数，两侧口径必须对齐，否则就是拿苹果比橘子。

诚实地摆开：循环版单是标准化那一步，就在 Python 这一层显式做了 $18 = 6 \times 3$ 次逐标量赋值（程序真实自增计数出来的，就是打印的 `n_scalar_loop`）；如果把前面求均值、求标准差的两组双层循环也算上，还要再加 $2 \times 18 = 36$ 次，合计约 $54$ 次 Python 层标量操作。向量化版这一侧，整个标准化只用了 **$4$ 个数组级操作**：`mean`、`std`、一次广播减、一次广播除（这也是 `n_array_ops_vec = 4` 的出处，不是拍脑袋的"$1$"）。数量级的对照才是要害：循环版的 Python 层操作数**与 $nd$ 成正比**，向量化版是**与矩阵大小无关的常数（$O(1)$）**。在 $6 \times 3$ 的小矩阵上是 $54$ 对 $4$；换成真实的 $10^5 \times 10^3$，循环版要在 Python 解释器里绕上 $\sim 10^8$ 圈，向量化版依然是那 $4$ 个数组级操作——底下的 $10^8$ 次乘除全被压进编译好的 C 里。这个"Python 层绕圈数"的差，就是常数因子的差，就是向量化的胜利。计时你想跑可以跑，那是一道辅助的风景；但**断言的证据是操作次数，不是秒数**。

最后看那两行"≈0"的输出，它把第二节埋的种子逼了出来。我没有直接打印列均值的具体数字，而是打印了 `np.allclose(Z_vec.mean(axis=0), 0.0)` 得到 `True`——这是有意为之。标准化后列均值在**数学上**恰是 $0$，但**机器上**是 $10^{-16}$ 量级的浮点残渣（这台机器上跑出来是 $[-4.4\times10^{-16},\ 5.6\times10^{-17},\ 7.4\times10^{-17}]$）。这些残渣的具体数值、乃至它四舍五入后显示成 `-0.` 还是 `0.` 的正负零符号，会随平台、随 BLAS 后端而微微不同，**不该被当成"人人一字不差"的东西去核对**。真正稳定、可断言的，是"它足够接近 $0$"——所以我们从头到尾用 `np.allclose` 而非 `==`。机器的 $0$ 不是数学的 $0$，这是本章要你带走的一根刺。

### 广播的威力，以及它的代价（进阶）

主线到这里已经闭环。但广播的能耐远不止"每行减均值"这种二维小把戏，我想再给你看一眼它最漂亮、也最危险的用法——把一个双重循环整个蒸发掉。这一段是 stretch，跨度比前面大，第一次读可以先跳过，回头再来。

任务是算**成对欧氏距离矩阵**：$n$ 个样本两两之间的距离，结果是一个 $(n, n)$ 的矩阵。下面这段接着上面的脚本跑（`X`、`n` 已定义）：

```python
# 接着上面的脚本（X, n 已定义）

# ---------- 广播形状小实验 ----------
a = np.arange(3).reshape(3, 1)             # (3,1)
b = np.arange(4).reshape(1, 4)             # (1,4)
print("(3,1)+(1,4) 广播成:", (a + b).shape)

# ---------- 成对欧氏距离：一个双重循环蒸发成三行 ----------
diff = X[:, None, :] - X[None, :, :]       # (6,1,3)-(1,6,3) -> 物化出真实的 (6,6,3)
Dmat = np.sqrt((diff ** 2).sum(axis=2))    # (6,6) 距离矩阵，无一显式循环
Dloop = np.zeros((n, n))
for i in range(n):
    for j in range(n):
        Dloop[i, j] = np.sqrt(((X[i] - X[j]) ** 2).sum())
print("成对距离 allclose:", np.allclose(Dmat, Dloop))
print("Dmat[0]:", np.round(Dmat[0], 4))
```

输出：

```
(3,1)+(1,4) 广播成: (3, 4)
成对距离 allclose: True
Dmat[0]: [0.     4.668  2.1389 4.4834 4.3972 5.0045]
```

`X[:, None, :]` 用 `None` 插进一个长度为 $1$ 的新轴，把形状撑成 $(6,1,3)$；`X[None, :, :]` 撑成 $(1,6,3)$。两者相减，广播在前两个轴上各自拉伸，得到 $(6,6,3)$——第 $(i,j)$ 个位置正好是样本 $i$ 减样本 $j$ 的差向量，平方、沿最后一个轴求和、开根，一个双重循环就此蒸发成三行没有循环的表达式。

但这里藏着第二节那个"假装、不占 $n$ 倍内存"说法的重要但书，务必看清：广播让**被拉伸的操作数**保持虚拟（`X` 那两份没有真的各复制 $6$ 遍），**可是相减的结果 `diff` 是一个实打实物化在内存里的 $(6,6,3)$ 张量**，占 $O(n^2 d)$ 的空间。在 $6$ 个样本上这无关痛痒；在 $10^5$ 个样本上，$(10^5, 10^5, d)$ 会瞬间撑爆内存。**广播省的是操作数的复制，不是结果的存储**——把这两件事混为一谈，是从小脚本走向真实数据时最容易踩的坑，也正是将来要引入分块计算、专用距离核（如 `scipy` 的 `cdist`）的动机。漂亮的手法要连着它的代价一起记住。

## 四、研究线（The Where）：思维活在张量里

把镜头拉远。你刚刚用 NumPy 做的事——把数据装进张量、用广播和整体运算操纵它——**不是 NumPy 的一个特性，而是整个现代深度学习的共同底座。** PyTorch、JAX、TensorFlow，这些框架你可以先粗暴地理解成一句话：**它们是"会自动求导的 NumPy"。** 张量、形状、广播、逐元素与线性代数运算，几乎原封不动地搬了过去；它们额外加的那件大事，是能对你写的这一整串张量运算**自动求梯度**（automatic differentiation）。这也是为什么本章不是可选的热身——你在这里养成的"用张量思考"的头脑，就是你将来读得懂、写得动那些框架的头脑。等到 Book IV，我们会亲手实现一个麻雀虽小五脏俱全的 micro-autograd，把"会自动求导的 NumPy"这句话从比喻变成你自己写的代码。那是全书的一个工程里程碑，而它的地基就在本章。

研究线还有一件更严肃的事，关乎科学诚信：**可复现性 (reproducibility)。** 你注意到脚本第一行 `default_rng(42)` 那个 `42` 了吗？它是随机种子。有了它，`rng.normal` 抽出的那串数，序列是**完全确定**的——所以我上面才敢把 `Dmat[0]` 这类数字四舍五入到几位印在正文里让你核对。

但要说得比很多教材更诚实一点，因为这句话和本章自己的浮点论点必须自洽：**固定种子给你的是"抽样序列的确定性"，不是"跨任何机器逐位相同的浮点结果"。** `default_rng` 底下的 PCG64 生成的整数流跨平台确定；可一旦这些数进了 `mean`、`std`、`@` 这些走 BLAS 内核的运算，不同平台、不同 BLAS 实现可能在末位比特上给出微小差异。这正是为什么可复现性靠三件事一起撑着——**固定随机种子、追求确定性、锁定环境**（库的版本、依赖、后端）——那第三件"锁定环境"恰恰承认了跨环境会有末位差异。所以正文里印的是舍入后的值供你核对，而全书判断"两版一致"用的始终是 `np.allclose`（容差内相等）而非 `==`（逐位相等）。可复现，是"在数值容差内可被别人重现"，不是"宇宙范围内逐比特相同"——这个区分本身就是数值思维的一部分。**能被别人复现，是一个结论从"我觉得"升级为"确实如此"的分水岭。**

最后，按体例，给你一个**可被检验、可被证伪的研究式问题**，请把它当真：

> 我们断言循环版和向量化版的标准化"结果一致"，但 `assert` 只测了种子 `42` 生成的那**一个**输入。这两版实现真的在**所有**输入上等价吗？如何用**性质测试 (property-based testing)**（比如 Hypothesis 这类工具）自动生成成千上万个随机形状、随机数值的矩阵——包括含 `NaN`、含常数列（标准差为 $0$，会触发除零）、含极大极小值的刁钻输入——去**证伪**"两版等价"这个命题？顺带说，我们上面回避了的那个"常数列除零"之问，正会在这里现形：$\sigma_j = 0$ 时，循环版会抛 `ZeroDivisionError`，向量化版则得到 `nan`/`inf`——**两版行为不再一致**。如果找到一个让两版不一致的输入，那个不一致到底源于这种逻辑/边界差异，还是仅仅源于浮点累加顺序不同带来的、在 `allclose` 容差之内/之外的数值差异？

能把这个问题问清楚、答明白，你就同时握住了软件工程的严谨和数值计算的诚实——这两样，是研究者和"能跑就行"的人之间的分界线。

## 七、回到中心对象：我们现在能亲手驾驭计算了

回到贯穿全书的中心对象：**智能是在不确定性与资源约束下、从经验中获得达成目标的能力；而学习，是用数据做归纳推断。**

第 3 章我们理解了"计算"这个词的能力边界与代价——那是**知**。这一章，我们第一次真正把手放到了计算上：用张量给"数据"（中心对象里那个词、第 1 章那个 $X$）造出了实实在在的容器，用向量化整块地操纵它而不必一次搬一块砖，用断言与随机种子让这份操纵可被检验、可被复现。一句话收束：**我们从此拥有的，是一种能被信任的、整块驾驭"数据容器"的能力。**

但请用中心对象的语言，逼出下一个非问不可的问题。我们很会操纵张量这个容器了——可我们从头到尾都在用 `rng.normal` 凭空**捏造**里面的数字。中心对象说学习是"用**数据**做归纳推断"，可数据从哪来？**世界不是天生就长成 $(n, d)$ 矩阵的样子。** 一段文字、一张脸、一次点击、一个分子——它们怎么才能变成能装进张量的数字？该往这个容器里装什么、又该怎么装，才不至于在装的那一刻就悄悄丢掉了智能所依赖的信息？

容器已经就位，手已经练好。第 5 章，我们就去回答这个更靠上游、也更危险的问题：**《数据——把世界变成数字》。**
