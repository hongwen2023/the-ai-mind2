# 第 2 章 · 智能的多副面孔

在第 1 章，我们做了一件看似冒失的事：把"智能"这个庞然大物切下一片——"学习"——放到手术台上，用 $X,Y,\mathcal{D},\mathcal{H},\ell,R,\hat R$ 这一组符号把它形式化，直到"从数据中做归纳推断"这句话变成一个你可以在纸上推、在代码里跑的东西。你现在握着这组符号，也许会产生一个合理的怀疑：凭什么是学习？智能一定要从"数据"里长出来吗？人类工程师满脑子的智慧——棋理、逻辑、常识——难道不能直接**灌**进机器里吗？

这不是一个天真的问题。它恰恰是过去七十年里，这个领域真正吵过、并且用几十亿美元和几代人的职业生涯下过注的问题。本章我们后退一步，不急着往学习的纵深里走，而是先站到高处，看清楚：在"达成目标的能力从何而来"这个中心问题上，历史上给出过**三种根本不同的答案**。看清这三张面孔，你才会明白：全书选择把"学习"放在圆心，不是因为它时髦，而是因为它在一场有据可依的范式竞争里胜出了——而理解它为什么胜出、在哪里胜出、以什么代价胜出，比单纯接受结论重要得多。

## ① 概念主线：三种关于"能力从哪来"的下注

回到我们的中心对象：**智能，是在不确定性与资源约束下，从经验中获得达成目标的能力。** 请把注意力放在"能力从哪来"这半句上。假设你是一位 1960 年代的研究者，手里有一台算力可怜的机器，任务是让它"聪明"起来——比如，下好一盘棋、诊断一种病、走出一座迷宫。工程师的智慧是有限而宝贵的资源，你必须决定：**把这份智慧放进哪里？** 历史给出了三个答案，我称之为三种"下注"。

**第一注，搜索 (search)：能力 = 在可能性空间里搜索。** 这一派的世界观极其干净。任何"达成目标"的任务，都可以形式化为三样东西：一组**状态 (state)**、一个**目标测试 (goal test)**、以及一组把状态变成新状态的**算子 (operator)**。于是"解决问题"就等于：从初始状态出发，反复施加算子，在由此展开的巨大**状态空间**里，搜索出一条通向目标的路径。走迷宫是这样（状态=你所在的格子，算子=上下左右）；下棋是这样（状态=局面，算子=合法着法，目标=将死对方）；规划一次物流调度也是这样。这一派把工程师的智慧放进了**启发式 (heuristic)**——一个告诉搜索"往这个方向走更有希望"的估价函数，用它来剪掉不值得看的枝杈。智慧的形态，是"搜索时的品味"。

**第二注，逻辑/符号 (symbolic / logic)：能力 = 显式表示知识 + 推理。** 这一派的直觉更贴近我们对"懂"的日常理解。一个内科医生之所以会诊断，是因为他"知道"很多事实与规则：发烧且咳嗽且胸片有阴影，则怀疑肺炎；肺炎则用抗生素。那么，把这些知识用一种机器能操作的形式（逻辑公式、产生式规则）**显式地写下来**，再配一台推理机在其上做演绎，机器不就会诊断了吗？这就是**知识表示 (knowledge representation)** 与**专家系统 (expert systems)** 的纲领。它把工程师的智慧放进了**知识库**：智慧的形态，是"被写下来的规则"。它的野心是惊人的——直接编码人类的理性；它的脆弱，我们稍后会看到，也同样惊人。

**第三注，学习 (learning)：能力 = 从经验/数据中归纳。** 这正是第 1 章形式化的那一套。它的下注方式是最反直觉、也最"偷懒"的：工程师**既不**亲手写下搜索的品味，**也不**亲手写下诊断的规则，而是退一步，只做两件事——划定一个假设空间 $\mathcal{H}$、给出一个损失 $\ell$，然后让**数据**去从 $\mathcal{H}$ 里挑出那个经验风险最小的假设。工程师的智慧被放进了**归纳偏置**（体现为 $\mathcal{H}$ 的形状）与**数据**本身。智慧的形态，是"我们选择让机器从什么里、以什么标准去归纳"。

请务必看清：这三者不是三个不相干的技术选项，井水不犯河水。它们是对**同一个问题的三种争抢**——工程师那份宝贵而有限的智慧，到底该注入**搜索算子里**、**知识库里**、还是**数据与损失函数里**？

这是一场零和的架构抉择——但请精确地理解这个"零和"，它日后会救你于混乱。**它零和的粒度，是单个组件。** 对某一个具体的能力（比如"如何评估一个局面到底好不好"），那份有限的工程智慧只能注入一处：评估函数要么手写、要么学习，你不可能同时既手写又学习同一个 $f$。可是，一台**完整的系统**，完全能够在不同组件上拼装不同的范式——我们在研究线会看到，Deep Blue 是"搜索 + 符号"，AlphaGo 是"搜索 + 学习"。所以零和的是"**一个组件的智慧注入位置**"，而不是"一台机器只准信一种教"。想清楚了这层区分，你就有资格去问那个真正的问题：为什么在决胜的那个组件上，第三注赢了？

## ② 数学线：把"搜索为何撞墙"讲成一个可量化的事实

要理解这场竞争，你不需要很多定理，但你需要一个数——一个会指数增长的数。让我们把"搜索"这一注做最小的形式化，因为它撞墙的方式最干净、最可量化，而它撞的那堵墙，恰恰是逼出后两注的原因。

把状态空间看成一张**图** $G=(V,E)$：节点 $v\in V$ 是状态，有向边 $(u,v)\in E$ 表示"存在一个算子把 $u$ 变成 $v$"。于是"求解"就是在 $G$ 上，从起点找一条到达目标节点集合的**路径**。这个抽象干净得让人愉悦：迷宫、棋局、规划，全都塌缩成"图上找路"。

麻烦出在图的**大小**上。设每个状态平均能展开出 $b$ 个后继（称 $b$ 为**分支因子 (branching factor)**），而你需要向前看 $d$ 步（**深度 (depth)**），那么要穷举的节点数大约是

$$
N \;\approx\; b^0 + b^1 + \cdots + b^d \;=\; \frac{b^{d+1}-1}{b-1}\;=\;\Theta(b^d).
$$

关键不在这个求和公式，而在那个 $b^d$：它是**指数**。这就是**组合爆炸 (combinatorial explosion)**——搜索范式的原罪。这里必须先分清两个极易混淆的量：一个是**可达局面数**（状态空间里有多少个不同的合法局面），另一个是**博弈树的大小**（把每条不同的着法路径都算成一片不同的叶子）。后者通常远大于前者，因为同一个局面可以经由许多不同的着法顺序到达，会被博弈树重复计数——这个区分，等下在习题里会变成一道专门的题。让我们把 $b^d$ 变成具体的数，你会立刻感到那堵墙：

- **井字棋 (tic-tac-toe)**：$b\le 9$、$d\le 9$，整棵博弈树的叶子上界是 $9! = 362{,}880$，可达局面仅 $5{,}478$ 个。这是一个**可以完全穷举**的世界。
- **国际象棋 (chess)**：合法局面数约 $10^{43}\sim10^{44}$（Shannon 1950 本人估计约 $10^{43}$，Tromp 给出的上界约 $4.8\times10^{44}$）；而整棵**博弈树**的规模——即著名的 **Shannon 数**——高达约 $10^{120}$。这个 $10^{120}$ 已经超出宇宙中原子总数（$\sim 10^{80}$）的想象半径。（下文用常数分支因子估出的 $35^{80}\approx 10^{123}$，与 Shannon 的 $10^{120}$ 是**同一个量——博弈树复杂度——的两种粗近似**，不必纠结哪个更"对"。）
- **围棋 (Go)**：合法局面数约 $10^{170}$。这不是"更大"，这是另一个宇宙。

同一个 $b^d$，同一套搜索算法，在第一个世界里**完胜**（下文你会亲手验证它下出不败之棋），在后两个世界里**瞬间蒸发**。请把这句话刻下来：**纯搜索之所以撞墙，不是因为它错了，而是因为指数不给任何有限的算力留活路。** 这是一个可以量化、无从辩驳的事实，而不是一句哲学感慨。（至于"$b^d$ 到底意味着多少不可承受的时间与空间"、指数与多项式之间那道深渊般的界线，是第 3 章《计算的能力与代价》的正题；这里我们先把这个指数当作一记警钟。）

现在，请你戴上第 1 章的眼镜再看一遍这三注，你会发现一条贯穿始终的暗线：**三种范式，其实都在与同一件事角力——如何在有限资源下注入归纳偏置。**

- **搜索**用**启发式**注入偏置：启发式函数告诉搜索"哪个方向更可能通往目标"，从而在指数的枝杈里只走极小一撮。好的启发式，就是把 $b^d$ 里那个 $b$ 有效地压小。
- **符号**用**规则**注入偏置：规则库直接规定了"什么可以推出什么"，把无边的可能性剪裁成一小片被允许的推理。
- **学习**用 $\mathcal{H}$ **与数据**注入偏置：第 1 章讲过，没有归纳偏置就没有泛化；$\mathcal{H}$ 的选择就是你押给这个世界的先验形状，数据则是让这个先验落地的经验。

三注争的从来不是"要不要偏置"——没有偏置寸步难行——争的是**偏置由谁提供、以什么形式提供**：由工程师手写（搜索的启发式、符号的规则），还是由数据在一个被工程师划定的空间里自己长出来（学习）。看清这条暗线，你就握住了本章的枢纽。

诚实的边界：本章不追求定理密度。严格的复杂度分析留给第 3 章，学习理论的正式机器留给 Book II。此刻我们只需要那个指数，和那条"三注都在与资源约束和归纳偏置角力"的主线。

## ③ 工程线：亲手感受"搜索的胜利与撞墙"

概念是廉价的，直到你让它在机器上跑起来。下面这段代码做两件事：第一，它用 **minimax + alpha-beta 剪枝**把井字棋**完全求解**，让你亲眼看到搜索在小世界里下出**完美不败**的棋；第二，它挂了一个**节点计数器**，让"搜索了多少"从一个抽象概念变成一个具体的数——你会看到这个数，以及它在换成大棋盘时如何注定失控。

这是一个从上到下可直接运行的单脚本，不依赖任何第三方库，没有随机性（因而完全确定，无需固定种子）：

```python
# 井字棋的完美博弈：minimax + alpha-beta，并统计搜索节点数
# 约定：X 为极大方(先手)，O 为极小方(后手)。空格=0，X=1，O=-1。

WIN_LINES = [
    (0,1,2),(3,4,5),(6,7,8),   # 行
    (0,3,6),(1,4,7),(2,5,8),   # 列
    (0,4,8),(2,4,6),           # 对角线
]

def winner(board):
    for a,b,c in WIN_LINES:
        s = board[a] + board[b] + board[c]
        if s == 3:  return 1     # X 三连
        if s == -3: return -1    # O 三连
    return 0

def full(board):
    return all(v != 0 for v in board)

# counter 是一个可变列表，用来记录被展开(评估)的节点数
def minimax(board, player, counter, alpha=-2, beta=2):
    counter[0] += 1
    w = winner(board)
    if w != 0:
        return w          # 终局：+1 = X 胜, -1 = O 胜
    if full(board):
        return 0          # 平局

    if player == 1:       # X 走：极大化
        best = -2
        for i in range(9):
            if board[i] == 0:
                board[i] = 1
                val = minimax(board, -1, counter, alpha, beta)
                board[i] = 0
                if val > best: best = val
                if best > alpha: alpha = best
                if alpha >= beta: break      # ← alpha-beta 剪枝
        return best
    else:                 # O 走：极小化
        best = 2
        for i in range(9):
            if board[i] == 0:
                board[i] = -1
                val = minimax(board, 1, counter, alpha, beta)
                board[i] = 0
                if val < best: best = val
                if best < beta: beta = best
                if alpha >= beta: break
        return best

def best_move(board, player, counter):
    """返回当前玩家的最优落子位置。"""
    best_val = -2 if player == 1 else 2
    best_i = -1
    for i in range(9):
        if board[i] == 0:
            board[i] = player
            val = minimax(board, -player, counter)
            board[i] = 0
            if player == 1 and val > best_val:
                best_val, best_i = val, i
            if player == -1 and val < best_val:
                best_val, best_i = val, i
    return best_i, best_val

def selfplay():
    """两个完美玩家对弈，返回(终局结果, 总展开节点数, 终局棋盘)。"""
    board = [0]*9
    counter = [0]
    player = 1
    while winner(board) == 0 and not full(board):
        i, _ = best_move(board, player, counter)
        board[i] = player
        player = -player
    return winner(board), counter[0], board

if __name__ == "__main__":
    import math
    # (1) 从空棋盘求解，评估先手 X 的博弈价值
    counter = [0]
    val = minimax([0]*9, 1, counter)
    print("空棋盘博弈价值(X先手):", val, "=> 0 表示双方完美时必和")
    print("完整求解展开的节点数(含alpha-beta剪枝):", counter[0])

    # (2) 两个完美玩家自我对弈
    result, nodes, final = selfplay()
    print("完美对弈结果:", {1:"X胜", -1:"O胜", 0:"平局"}[result])

    # (3) 无剪枝时博弈树的规模量级
    print("叶子数上界 9! =", math.factorial(9))
```

**运行后你会看到（逐行核验）：**

```
空棋盘博弈价值(X先手): 0 => 0 表示双方完美时必和
完整求解展开的节点数(含alpha-beta剪枝): 18297
完美对弈结果: 平局
叶子数上界 9! = 362880
```

三行断言，三个可以被你验证的结论，请一个一个咀嚼：

1. **博弈价值 = 0**：从空棋盘出发，minimax 递归到底、回溯到顶，算出的最优结果是 $0$——**平局**。这正是井字棋的著名事实：双方都完美时，必和。这个 $0$ 不是我们告诉机器的，是它把整棵博弈树看穿之后**自己算出来**的。这就是"搜索的胜利"：在一个可穷举的世界里，搜索给出的不是"很强的棋"，而是**证明意义上不可能被击败的棋**。

2. **节点数 = 18{,}297**：这就是"智慧的代价"被量化成一个数。有趣的是对照实验——如果你把 alpha-beta 剪枝去掉（去掉那两行 `if alpha >= beta: break`，并把 $\alpha,\beta$ 的传递删掉），同样一次完整求解要展开 **549{,}946** 个节点。剪枝把它压到 $18{,}297$，$549946/18297\approx 30.06$，**降了约 30 倍**——而剪枝的本质，正是"当我已经知道这条枝不可能比手上的更好时，就不必再看"。这就是**启发式/剪枝作为注入偏置的一次微缩现场**：我们没有更聪明地搜索每一个节点，我们只是聪明地**不看**大量节点。请把这一点连回概念主线：搜索派押注的全部技艺，浓缩起来就是这个 `break`。

3. **完美对弈 = 平局**，而叶子上界只有 $9! = 362{,}880$。整个世界小到一台手机毫秒级穷举。

**现在，把棋盘换掉，看墙如何砸下来——但请务必用同一把尺子去量三个世界。** 我们用**常数分支因子**的 $b^d$ 上界来量：井字棋 $b\approx 9$、$d\approx 9$，$9^9\approx 3.9\times10^{8}$（$10^{8.6}$ 量级）；国际象棋每步约 $b\approx 35$、一盘约 $d\approx 80$，$35^{80}\approx 10^{123}$；围棋 $b\approx 250$、$d\approx 150$，$250^{150}\approx 10^{360}$。（注意：井字棋因为分支因子**逐步递减**、且很多路径**提前终局**，真实的叶子上界比 $9^9$ 紧得多，正是我们上面用到的 $9!\approx 3.6\times10^{5}$；但为了公平对照，此处对三者一律用常数 $b$ 的上界。）你不需要把这些数算精确，你只需要感受那个断崖：**用同一把尺子，井字棋是 $10^{8}$，国际象棋是 $10^{123}$，围棋是 $10^{360}$。** 同一段 `minimax` 代码，把 `range(9)` 换成一张 $19\times19$ 的棋盘，它**在算法上完全正确，却永远不会返回**——不是几分钟、几小时，而是超过宇宙年龄若干个数量级。

这就是这一节要你亲手摸到的东西：**纯搜索在井字棋上完胜，在围棋上指数撞墙，而中间没有平滑的过渡，只有一道由 $b^d$ 划出的悬崖。** 于是问题被逼到墙角：既然不能把整棵树看到底，机器又如何判断一个**没下完**的局面到底是好是坏？

答案只有一条路：你需要一个**评估函数 (evaluation function)** $f(\text{局面})\to \mathbb{R}$，它不必看到终局，就能估出"这个局面对我有多有利"，让搜索在浅得多的深度上截断。问题随之变成：**这个 $f$ 从哪来？** 这里，历史给出了它最深刻的一次分岔——而这道分岔，正好就是三注之争的决胜局，我们留到研究线去看。

## ④ 研究线：苦涩的教训，与"推理还是检索"的当代战场

上一节末尾那个悬而未决的 $f$，把我们直接带到这个领域最著名、也最有争议的一段观察面前。本节分四步走，给你一根导航绳：**先看 $f$ 的两种来路（Deep Blue vs. AlphaGo），再把它抽象成一句判断（The Bitter Lesson），然后看这句判断如何被神经符号的回潮挑战，最后落到你能亲手测量的两个问题上。**

**第一步——Deep Blue vs. AlphaGo：同一个 $f$，两种来路。** 1997 年击败卡斯帕罗夫的 Deep Blue，它的评估函数 $f$ 是**人手工写的**——由国际象棋大师和工程师一起，把子力价值、棋子活动性、王的安全等等棋理，编码成一个复杂的加权公式。这是搜索派与符号派的一次联姻：用搜索处理博弈树，用**人类写下的知识**处理评估。（这不违反① 的零和断言——零和说的是**单个组件**的智慧从哪来，不是整机不能混用范式；Deep Blue 恰恰是"搜索管树、符号管 $f$"的跨组件拼装。）它赢了，但它赢的方式，是把顶尖人类棋理**灌**进机器。二十年后的 AlphaGo 走了另一条路：它的评估函数不是写的，是**学的**——用一个神经网络（严格说是策略网络 + 价值网络，其中价值网络承担评估 $f$ 的角色），从海量对局（先是人类棋谱，后是自我对弈）中，把"这个局面有多好"这件事**归纳**出来。围棋 $10^{170}$ 的状态空间，人类写不出、也不可能写出一个够用的 $f$；于是唯一的出路，是让 $f$ 从数据里长出来。**从 Deep Blue 到 AlphaGo，是评估函数从"手写"到"学习"的历史转折——也正是本书把"学习"放在圆心的现实注脚。** 请注意，它并没有抛弃搜索：AlphaGo 依然在做树搜索（蒙特卡洛树搜索），只是搜索所依赖的那个"品味"$f$，交给了学习。搜索 + 学习，联手绞杀了纯符号那一注。

**第二步——The Bitter Lesson。** Rich Sutton 在 2019 年把这段历史提炼成一篇短文《苦涩的教训》，论点锋利到近乎冒犯：回顾 AI 七十年，凡是研究者试图把**人类知识**精巧地嵌进系统的努力，长期看几乎都输给了那些**朴素地、更充分地利用算力**的通用方法——而利用算力的通用方法，归根结底就是两样：**搜索**与**学习**。国际象棋如此，围棋如此，语音识别、计算机视觉、自然语言，一次又一次如此。之所以"苦涩"，是因为它否定了研究者最珍视的东西——我们的领域知识、我们的巧思——它说：随着算力增长，这些精巧的人类知识不但没帮上忙，反而成了天花板。

请以研究者的姿态对待这个论断，而不是信徒的姿态。**它是一个有力但有争议的经验观察，不是定理。** 它的反对者会立刻指出：算力也不是无穷的；在数据稀缺、安全攸关、需要可解释的领域，嵌入人类结构常常是唯一务实的选择；而且，哪怕是最"通用"的方法，其架构本身（卷积、注意力）不也是人类知识的结晶吗？"少即是多"和"结构即偏置"之间的张力，至今没有被判决。你要记住的是它的形状：它是一个关于**趋势**的赌注，而不是一条关于**每个具体问题**的铁律。

**第三步——神经符号的回潮。** 有趣的是，被苦涩教训判了缓刑的符号那一注，并没有死。**神经符号 (neuro-symbolic)** 方法正在回潮：让神经网络负责感知与归纳、让符号系统负责严格的、可组合的推理，试图各取所长。这正是① 里"跨组件拼装"的又一个现场——感知归组件交给学习，推理组件交给符号。而这背后，是当下这个领域最灼热、也最能检验的一个问题：

> **第四步——大语言模型（LLM），到底是在"推理"，还是在"检索/模式匹配"？**

当一个模型解出一道它从没见过的数学题，它是真的在进行某种**推理搜索**，还是在从训练语料里检索出一个足够接近的模式做插值？更微妙的是，近两年兴起的 **test-time compute / 思维链 (chain-of-thought)**：让模型在给出答案前，先生成一长串中间步骤、甚至在多条推理路径间做选择与回溯——这看起来惊人地像是"搜索"以一种新形式，在语言空间里悄悄回归。三注之争，似乎在最新的技术里重新洗牌了。

这不是清谈，你可以把它变成**可检验的问题**。给你两个：

1. 若一个模型是"检索/模式匹配"而非"推理"，那么当我把一道题的**表面特征**（变量名、数字、叙事包装）替换，但**保持逻辑结构不变**时，它的正确率应当显著下降；若是真推理，则应当稳健。设计一组这样的"同构变体"题库，测量正确率随表面扰动的衰减曲线——这是一个可以真做的实验，而它的结论会直接落在"推理 vs. 检索"的天平上。
2. 若思维链是"搜索的回归"，那么给模型更多 test-time compute（更长的推理链、更多采样路径）应当在**搜索友好**的问题（有清晰的正确性判据、解空间可被步步逼近）上带来持续增益，而在**纯检索**问题上很快饱和。测这两类问题上增益随计算量的曲线，曲线的形状就是证据。

能把一个大问题化成这样一条**可测量的曲线**，你就从"读者"变成了"研究者"。这正是研究线要交给你的东西。

**Q2.** 分支因子 $b=35$、搜索深度 $d=80$ 时，博弈树节点数约为多少个数量级？这对"纯搜索下国际象棋"意味着什么？

**Q3.** 在井字棋 minimax 代码里，alpha-beta 剪枝把展开节点数从约 $55$ 万降到约 $1.8$ 万。请用"注入归纳偏置"的语言解释：剪枝到底做了什么？

**Q4.** Deep Blue 与 AlphaGo 都用了搜索，为什么说从前者到后者体现了"学习"范式的胜利？关键差别在哪个组件上？

**Q5.** 为什么说《苦涩的教训》是"观察"而非"定理"？举出一个它可能失效的情境。

**Q6.** 用一句话说清"可达局面数"与"博弈树大小"的区别，并解释为什么后者通常远大于前者。

<details>
<summary>参考答案（点击展开）</summary>

**A1.** 搜索：注入进**启发式/剪枝**（搜索时的品味）；符号：注入进**知识库/规则**（被写下来的知识）；学习：注入进**假设空间 $\mathcal{H}$ 与数据**（让机器从什么里、以什么损失去归纳）。

**A2.** $35^{80}\approx 10^{123}$（与 Shannon 数 $10^{120}$ 是同一个量——博弈树复杂度——的两种粗近似）。远超宇宙原子数 $10^{80}$。意味着纯穷举搜索在国际象棋上**在物理上不可能完成**，必须靠评估函数在浅深度截断——而评估函数从哪来，正是范式分岔点。

**A3.** 剪枝在利用一条偏置：**"当前枝已被证明不可能优于手上最好结果时，其后续无需再看。"** 它没有更聪明地评估每个节点，而是聪明地**跳过**大量节点。这是把"极大极小的对抗结构"这一先验知识，转化为对搜索空间的裁剪——是搜索派注入归纳偏置的最小现场。

**A4.** 关键差别在**评估函数 $f$**。Deep Blue 的 $f$ 由人类棋理**手工编码**；AlphaGo 的 $f$（价值网络）由神经网络从对局数据中**学习**得到。围棋 $10^{170}$ 的状态空间使人写 $f$ 不可行，唯有学习。二者都搜索，但"搜索所依赖的品味从哪来"从手写变成了学习——这正是学习范式的胜利。

**A5.** 因为它是对历史**趋势**的经验归纳，没有证明、有反例风险。可能失效的情境：数据极度稀缺、需要可证明的正确性或可解释性、安全攸关、或算力受限的边缘设备——此时嵌入人类结构往往优于"堆算力"。

**A6.** "可达局面数"数的是有多少个**不同的合法局面**；"博弈树大小"数的是有多少条**不同的着法路径**（叶子）。后者远大于前者，因为同一个局面能经由许多不同的着法顺序到达，会在博弈树里被反复重复计数——这也提示：搜索时对重复局面做**换位表 (transposition table)** 缓存，能省下大量重复展开。

## ⑥ 白板自检清单

关掉书，拿一支笔，逐条问自己"我能不能当场做到"。注意：⑤ 是在**调取事实**，这一节要的是**当着一个同伴、把一整条推理链讲出来**——能讲通，才算真懂。

- [ ] **数学线：** 我能凭空写出 $N\approx b^d$，并向同伴讲清为什么"指数"意味着搜索必然在某个规模撞墙；我能顺带说清它的失效边界——即"什么情况下 $b^d$ 其实不可怕"（比如启发式把有效 $b$ 压到接近 1，或问题深度极浅）。
- [ ] **工程线：** 我能从零把井字棋的 minimax 写出来，让它下出必和的完美棋，并当场解释 alpha-beta 的 `break` 在剪掉什么、为什么剪掉是**安全**的（不改变最优值）。
- [ ] **研究线：** 我能一口气讲出《苦涩的教训》的论点、它的证据（Deep Blue→AlphaGo）、以及至少一条对它的合理反驳；我能提出一个区分"推理 vs. 检索"的可检验问题，并预测两种假设的不同结果。
- [ ] **概念线（含和解）：** 我能把三种范式讲成对"能力从哪来"的三种下注，用"归纳偏置注入到哪里"一句话把它们统一起来，**并且**当同伴反问"那 Deep Blue、AlphaGo 不是同时用了两注吗"时，我能立刻讲清：零和的是**单个组件**的注入位置，而完整系统可以跨组件混用范式——两者并不矛盾。
- [ ] **贯通：** 我能解释，为什么"学习"最终成为主流不是审美选择，而是被 $b^d$ 这堵墙和"评估函数从哪来"这个问题逼出来的结果。

任何一条讲不通，就回到对应的段落——理解是慢而不可逆的，补上再走。

## ⑦ 回到中心对象，与埋下的钩子

我们的中心对象说：智能是**在不确定性与资源约束下，从经验中获得达成目标的能力**。本章做的，是把这句定义里"能力从哪来"这半句，拆成历史上三张面孔——三种下注。搜索押注"能力=在可能性空间里搜索"，把智慧放进启发式；符号押注"能力=显式知识+推理"，把智慧放进规则；学习押注"能力=从经验中归纳"，把智慧放进 $\mathcal{H}$ 与数据。而它们争的，是同一件事：**那份有限的工程智慧，该注入到哪个组件里。** 全书把圆心放在"学习"，不是因为另外两注错了——搜索至今活在每一次 test-time 推理里，符号正以神经符号的形态回潮，一台真实系统常常同时用上两注——而是因为在"评估函数从哪来"这个决胜的**组件**上，数据一次次战胜了手写。这是第 1 章那套 $\mathcal{H},\ell,\hat R$ 之所以配坐圆心的历史正当性。

但请回看本章真正的枢纽：三注全都在与**资源约束**角力。这四个字，正是中心对象定义里我们**摸到了形状、却还没有理论化**的那个词。整条 $b^d$ 撞墙的论证，其实就是我们伸手去摸"资源约束"这堵墙——我们已经摸到它是**指数形状**的了，浓缩成一个符号就是 $b^d$。可是"摸到形状"离"真正理解"还差着一整章：指数增长到底意味着什么？它和多项式增长之间，隔着的是量的差别，还是**质的鸿沟**？为什么有些问题哪怕算到宇宙热寂也解不完，而另一些一眨眼就完？"资源约束"里的"计算"，到底遵循什么样的铁律？

在把这堵墙**理论化**之前——也就是在读懂"计算的能力与它的代价"之前——我们对"资源约束"的理解就还是残缺的；而没有这一半，"在资源约束下获得能力"这句中心定义就只是一句漂亮话。这正是第 3 章《计算的能力与代价》要交割的另一半。我们将把本章那个吓人的 $b^d$，放进**计算复杂度**的显微镜下，弄清 $P$ 与 $NP$、可解与不可解之间那道决定了"什么问题原则上够得着"的分界线。搜索撞的那堵墙，到那时，会从一记警钟，变成一张可以被精确测量的地图。

## 习题

**数学题**

**T1（★☆☆）.** 四子棋（Connect Four）标准棋盘为 $7$ 列 $\times 6$ 行。取分支因子上界 $b=7$、一盘最长 $d=42$ 手。(a) 估算其博弈树规模的数量级 $b^d$。(b) 已知其合法局面数约为 $4.5\times10^{12}$，而 $7^{42}\approx 10^{35}$，请解释为何"博弈树规模"远大于"合法局面数"——这个差距来自哪里？它提示我们优化搜索时可以利用什么？

**T2（★★☆）.** 设一个搜索问题分支因子恒为 $b$、目标在深度 $d$。(a) 推导广度优先搜索在最坏情况下展开的节点数关于 $b,d$ 的表达式。(b) 若我们有一个"完美"启发式，能把有效分支因子从 $b$ 降到 $b'=1.2$，当 $b=35,d=15$ 时，节点数从多少降到多少？用这个对比定量说明"启发式=注入偏置"的威力。

**T3（★★★）.** 已知井字棋含 alpha-beta 剪枝的完整求解展开约 $1.8\times10^4$ 节点，无剪枝约 $5.5\times10^5$。理论上，alpha-beta 在**最优着法排序**下可把需展开的叶子数从 $O(b^d)$ 降到 $O(b^{d/2})$。请解释这个 $b^{d/2}$ 意味着"有效分支因子"变成了 $\sqrt{b}$，并说明：为什么这**仍然不能**拯救围棋级别的搜索？（提示：把 $b^{d/2}$ 代入围棋的 $b=250,d=150$ 再看量级——$\sqrt{250}\approx 16$，$16^{150}$ 是多少？）

**编程题**

**P1（★☆☆）.** 在正文的 minimax 代码上，给 `minimax` 增加一个 `depth` 参数并让评分随深度衰减（如胜利返回 `10 - depth`、失败返回 `depth - 10`），使机器在必胜时**尽快**取胜、必败时**尽量**拖延。验证在对手第一步走角、你走中心时的着法变化，并报告节点数是否改变。

**P2（★★☆）.** 正文给出了"有剪枝"版本。请另写一个**无 alpha-beta** 的 `minimax`，在同一批初始局面上分别统计两者的展开节点数，复现"约 $30$ 倍"（正文实测为 $549946/18297\approx 30.06$）的差距。然后打乱着法的遍历顺序（把 `range(9)` 换成一个随机排列，**固定随机种子**），观察 alpha-beta 的节点数如何随着法排序的好坏而变化——这印证了 T3 里"着法排序决定剪枝效率"的论断。

**P3（★★★）.** 把棋盘从 $3\times3$ 井字棋换成 $4\times4$ 井字棋（四连获胜），复用同一套 minimax 框架，并加上节点计数。(a) 报告完整求解需要展开多少节点。(b) 若你的机器在合理时间内跑不完，请**估算**它会展开多少节点，并解释这正是 $b^d$ 组合爆炸的一次亲身遭遇。(c) 提出一个能让它可解的改动（如限制搜索深度 + 一个手写评估函数），并讨论：你刚刚做的，是把智慧从"搜索"挪向了哪一注？

**思考题**

**T4（开放）.** 当今的大语言模型，在你看来更像三注中的哪一注——搜索、逻辑，还是学习？请分别为"它是学习""它在做搜索（思维链/test-time compute）""它内隐了某种符号推理"各找一条支持证据和一条反驳证据。你不必给出最终判决，但要让每一条证据都是**可被检验**的，而非印象。

- **Russell & Norvig，AIMA 关于知识表示与一阶逻辑的章节。** "符号"一注的正典：看清专家系统的野心，以及知识获取瓶颈、常识与组合爆炸如何把它逼入墙角。
- **Rich Sutton，"The Bitter Lesson" (2019)。** 一篇短到可以一口气读完、却重塑了无数人研究品味的博文。请把它当成一个**有争议的论断**去读，一边读一边在页边写下你的反驳。
- **Silver et al., "Mastering the game of Go with deep neural networks and tree search," Nature (2016)；及 "Mastering the game of Go without human knowledge" (AlphaGo Zero, 2017)。** 亲眼看"评估函数从手写到学习"这场转折的原始证据；第二篇尤其能让你体会"连人类棋谱都不要、纯自我对弈"意味着什么。
- **Campbell, Hoane & Hsu, "Deep Blue," Artificial Intelligence (2002)。** 手工评估函数 + 深度搜索的巅峰之作；与 AlphaGo 对照读，你会对"智慧注入到哪里"这个抉择产生切肤之感。
- **Wei et al., "Chain-of-Thought Prompting Elicits Reasoning in Large Language Models" (2022)。** 当代"搜索是否以新形式回归"这一问题的起点文献之一；读它，再回看本章研究线里那两个可检验问题，你会知道该去测什么。
